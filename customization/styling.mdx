## Overview

WebEditor uses a combination of CSS variables, Tailwind CSS, and theme tokens to provide a consistent, customizable design system. The editor supports light and dark modes out of the box.

## CSS Variables

WebEditor's color system is built on CSS variables defined in `styles.css`. These variables use the OKLCH color space for better color consistency.

### Core Color Tokens

```css
:root {
  --radius: 0.625rem;
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  --card: oklch(1 0 0);
  --card-foreground: oklch(0.145 0 0);
  --popover: oklch(1 0 0);
  --popover-foreground: oklch(0.145 0 0);
  --primary: oklch(0.205 0 0);
  --primary-foreground: oklch(0.985 0 0);
  --secondary: oklch(0.97 0 0);
  --secondary-foreground: oklch(0.205 0 0);
  --muted: oklch(0.97 0 0);
  --muted-foreground: oklch(0.556 0 0);
  --accent: oklch(0.97 0 0);
  --accent-foreground: oklch(0.205 0 0);
  --destructive: oklch(0.577 0.245 27.325);
  --border: oklch(0.922 0 0);
  --input: oklch(0.922 0 0);
  --ring: oklch(0.708 0 0);
}
```

### Dark Mode

```css
.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
  --card: oklch(0.205 0 0);
  --card-foreground: oklch(0.985 0 0);
  --primary: oklch(0.922 0 0);
  --primary-foreground: oklch(0.205 0 0);
  --border: oklch(1 0 0 / 10%);
  --input: oklch(1 0 0 / 15%);
  /* ... */
}
```

### Tailwind Integration

CSS variables are mapped to Tailwind colors:

```css
@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-card: var(--card);
  --color-card-foreground: var(--card-foreground);
  --color-primary: var(--primary);
  --color-primary-foreground: var(--primary-foreground);
  --color-border: var(--border);
  /* ... */
}
```

## Component Styling

### Using Tailwind Classes

All components use Tailwind utility classes for styling:

```tsx
<div className="border border-border bg-card shadow-sm rounded-lg">
  <h6 className="text-card-foreground text-lg font-semibold">
    Card Title
  </h6>
</div>
```

### Color Classes

| Token | Usage |
|-------|-------|
| `bg-background` | Main background |
| `text-foreground` | Primary text |
| `bg-card` | Card backgrounds |
| `text-card-foreground` | Card text |
| `bg-primary` | Primary buttons |
| `text-primary-foreground` | Primary button text |
| `bg-secondary` | Secondary elements |
| `text-muted-foreground` | Muted/secondary text |
| `border-border` | Borders |
| `bg-accent` | Accent backgrounds |
| `text-destructive` | Error states |

### Border Radius

The editor uses a consistent border radius system:

```css
--radius-sm: calc(var(--radius) - 4px);
--radius-md: calc(var(--radius) - 2px);
--radius-lg: var(--radius);
--radius-xl: calc(var(--radius) + 4px);
```

## Customizing Component Styles

### Override Component Defaults

Create a custom CSS file to override component styles:

```css
/* custom-editor.css */
.card-component {
  --card-bg: oklch(0.98 0 0);
  --card-border: oklch(0.85 0 0);
}

.card-component {
  background: var(--card-bg);
  border-color: var(--card-border);
}
```

### Dynamic Styling in Components

```tsx
const getCardClasses = (horizontal: boolean, isClickable: boolean) => {
  const baseClasses = `
    border border-border bg-card shadow-sm rounded-lg
  `;

  const hoverClasses = isClickable
    ? `cursor-pointer transition-transform duration-300
       hover:shadow-lg hover:scale-[1.01]
       hover:border-primary/50`
    : "";

  return `${baseClasses} ${hoverClasses}`;
};
```

## Brand Colors

Define your brand color:

```css
:root {
  --brand: #6699cc;
  --brand-foreground: oklch(0.985 0 0);
}
```

Use it in components:

```tsx
<div className="bg-[var(--brand)] text-[var(--brand-foreground)]">
  Branded Element
</div>
```

## Best Practices

<Callout type="tip">
  Always use CSS variables instead of hardcoded colors to ensure proper theme support.
</Callout>

<Card title="Styling Guidelines" icon="Palette">
  - Use Tailwind utility classes for consistency
  - Leverage CSS variables for theme-aware components
  - Test components in both light and dark modes
  - Use semantic color names (e.g., `primary`, `destructive`) not color values
  - Apply `transition` classes for smooth state changes
</Card>
