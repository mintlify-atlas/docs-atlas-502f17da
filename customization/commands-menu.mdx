## Overview

The commands menu appears when users type `/` in the editor. It provides quick access to blocks, components, and formatting options. The menu is fully customizable through the `command-system.tsx` configuration.

## Command Types

WebEditor supports three types of commands:

### Simple Commands

Commands that insert text or markdown:

```typescript
export interface SimpleCommand {
  id: string;
  name: string;
  description: string;
  icon: typeof Hash;
  write: string;
}
```

Example:

```typescript
{
  id: "h1",
  name: "Heading 1",
  icon: Heading1,
  description: "Big section heading",
  write: "# "
}
```

### Component Commands

Commands that insert custom components:

```typescript
export interface ComponentCommand {
  id: string;
  name: string;
  description: string;
  icon: typeof Hash;
  componentName: string;
}
```

Example:

```typescript
{
  id: "code",
  name: "Code",
  icon: Code,
  description: "Capture a code snippet",
  componentName: "code_snippet",
}
```

### Attribute Commands

Commands that prompt for attributes before inserting:

```typescript
export interface AttributeCommand {
  id: string;
  name: string;
  description: string;
  icon: typeof Hash;
  componentName: keyof typeof COMPONENT_ATTRIBUTES;
  requiresAttributes: true;
}
```

```typescript
{ separator: true }
```

## Helper Functions

### Filter Commands by Type

```typescript
import {
  getSimpleCommands,
  getAttributeCommands,
  getComponentCommands
} from "./command-system";

const simpleCommands = getSimpleCommands();
const attributeCommands = getAttributeCommands();
const componentCommands = getComponentCommands();
```

### Search Commands

```typescript
import { filterCommands } from "./command-system";

const results = filterCommands("heading");
// Returns commands matching "heading" in name, description, or id
```

### Type Guards

```typescript
import {
  isAttributeCommand,
  isComponentCommand
} from "./command-system";

if (isComponentCommand(command)) {
  // Insert component
  insertComponent(command.componentName);
} else if (isAttributeCommand(command)) {
  // Show attribute dialog
  showAttributeDialog(command.componentName);
}
```

## Adding Custom Commands

### 1. Simple Text Command

```typescript
const customCommand: SimpleCommand = {
  id: "signature",
  name: "Signature",
  icon: FileText,
  description: "Insert your signature",
  write: "\n\n---\nYour Name\n"
};

commandMenuSetup.push(customCommand);
```

### 2. Component Command

```typescript
const customComponent: ComponentCommand = {
  id: "video",
  name: "Video",
  icon: Video,
  description: "Embed a video",
  componentName: "video_embed",
};

commandMenuSetup.push(customComponent);
```

### 3. Attribute Command

First, define the attribute configuration:

```typescript
// In attribute-prompts.tsx
export const COMPONENT_ATTRIBUTES = {
  // ... existing configs
  custom_card: {
    title: "Create Custom Card",
    description: "Configure your custom card",
    fields: [
      {
        name: "title",
        label: "Title",
        type: "text" as AttributeFieldType,
        required: true,
      },
      {
        name: "color",
        label: "Color",
        type: "text" as AttributeFieldType,
        defaultValue: "blue",
      },
    ],
  },
};
```

Then add the command:

```typescript
const attributeCommand: AttributeCommand = {
  id: "custom-card",
  name: "Custom Card",
  icon: CreditCard,
  description: "Create a custom styled card",
  componentName: "custom_card",
  requiresAttributes: true,
};

commandMenuSetup.push(attributeCommand);
```

## Organizing Commands

Group commands logically with separators:

```typescript
export const commandMenuSetup: CommandItem[] = [
  // Basic formatting
  { id: "h1", name: "Heading 1", /* ... */ },
  { id: "h2", name: "Heading 2", /* ... */ },
  { id: "paragraph", name: "Text", /* ... */ },

  { separator: true },

  // Advanced components
  { id: "card", name: "Card", /* ... */ },
  { id: "callout", name: "Callout", /* ... */ },

  { separator: true },

  // Media
  { id: "code", name: "Code", /* ... */ },
  { id: "mermaid", name: "Mermaid", /* ... */ },
];
```

## Search and Filtering

The command menu automatically filters commands as users type:

```typescript
export function filterCommands(searchTerm: string): CommandItem[] {
  if (!searchTerm.trim()) {
    return commandMenuSetup;
  }

  const lowerSearch = searchTerm.toLowerCase();

  const filteredCommands = commandMenuSetup.filter((item) => {
    if ("separator" in item) return false;

    const command = item as Command;
    return (
      command.name.toLowerCase().includes(lowerSearch) ||
      command.description.toLowerCase().includes(lowerSearch) ||
      command.id.toLowerCase().includes(lowerSearch)
    );
  });

  return filteredCommands;
}
```

## Best Practices

<Card title="Command Menu Tips" icon="Lightbulb">
  - Keep command names short and descriptive
  - Use clear, action-oriented descriptions
  - Choose intuitive icons that match the command purpose
  - Group related commands with separators
  - Limit the number of commands to avoid overwhelming users
</Card>

<Callout type="tip">
  The command menu searches across the `name`, `description`, and `id` fields, so choose these values carefully for better discoverability.
</Callout>
