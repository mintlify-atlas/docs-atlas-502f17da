Parameter fields document function or API parameters with name, type, required status, and description.

## Usage

Insert a field using the `/` menu:

```
/field
```

Or use the component tag:

```
<field name="userId" type="string" required />
```

## Features

- **Parameter name** - Displayed in monospace font
- **Type badge** - Shows the parameter type (string, number, etc.)
- **Required/Optional** - Visual indicator
- **Description** - Full markdown support for parameter description
- **Click to edit** - All attributes editable in place

## Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `name` | string | `""` | Parameter name |
| `type` | string | `"string"` | Parameter type |
| `required` | boolean | `false` | Whether parameter is required |

## Editing Fields

In edit mode, click on any part of the field to open the editor:

1. **Name** - Click the parameter name
2. **Type** - Click the type badge
3. **Required** - Click the required/optional badge
4. **Description** - Click inside the description area

All attributes can be edited in a single modal.

## Implementation Details

### Node Spec

```typescript
// field.tsx:14-42
export const fieldNodeSpec: NodeSpec = {
  group: "block",
  content: "block+",
  attrs: {
    name: { default: "" },
    type: { default: "string" },
    required: { default: false },
  },
  selectable: true,
  parseDOM: [
    {
      tag: "field",
      getAttrs: (dom) => ({
        name: dom.getAttribute("name") || "",
        type: dom.getAttribute("type") || "string",
        required: dom.getAttribute("required") === "true",
      }),
    },
  ],
}
```

### Update Handler

```typescript
// field.tsx:224-250
const updateField = useEditorEventCallback(
  (view, { name: newName, type: newType, required: newRequired }) => {
    const { getPos, node } = nodeProps;
    const pos = getPos();
    
    const tr = view.state.tr;
    tr.setNodeMarkup(pos, undefined, {
      ...node.attrs,
      name: newName,
      type: newType,
      required: newRequired,
    });
    view.dispatch(tr);
    setModalOpen(false);
    setEditField(null);
  }
);
```

### Input Rule

```typescript
// field.tsx:353-376
export const addFieldRule = new InputRule(
  /^<field(?:\s+([^>]+))?(?:\s*\/>|\s*>)/,
  (state, match, start, end) => {
    const attrs = {
      name: "",
      type: "string",
      required: false,
    };
    
    if (match[1]) {
      const nameMatch = match[1].match(/name="([^"]*)"/); 
      if (nameMatch) attrs.name = nameMatch[1];
      
      const typeMatch = match[1].match(/type="([^"]*)"/); 
      if (typeMatch) attrs.type = typeMatch[1];
      
      attrs.required = /required(?:=true|\s|$)/.test(match[1]);
    }
    
    const field = schema.nodes.field.create(attrs, schema.nodes.paragraph.create());
    tr.replaceRangeWith(start, end, field);
    return tr;
  }
);
```

## Common Parameter Types

- `string` - Text values
- `number` - Numeric values
- `boolean` - True/false values
- `object` - Complex objects
- `array` - Array values
- `function` - Callback functions
- `Promise<T>` - Async return types
- Custom types - e.g., `User`, `Config`, `Options`
